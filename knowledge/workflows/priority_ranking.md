# ワークフローKnowledge — 優先度判定

### WF-PRIORITY-001: 優先度判定方針

**Rule ID:** WF-PRIORITY-001
**Status:** Draft (2026-07-02 Knowledge Integration v1で登録)

**Statement:**
案件の優先度は単一指標ではなく、納期接近度・リスクフラグ・粗利トレンドの組み合わせで判定する。重み付け基準は会社として未決定のため、暫定評価は納期接近度を主軸とする。

---

### WF-PRIORITY-GRAIN-002: 優先度判定の粒度制約

**Rule ID:** WF-PRIORITY-GRAIN-002
**Status:** Draft (2026-07-02 Knowledge Integration v1で登録)

**Statement:**
優先度判定は案件レベルのみで行う。顧客別・商品別の粒度には適用しない。
